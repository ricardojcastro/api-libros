Endpoints

Todos los endpoints responden con application/json. El servidor corre en el puerto 3000 localmente y en Render en producción.

GET /api/libros Retorna todos los libros disponibles ▼
Descripción

Devuelve el arreglo completo de libros almacenados en memoria. No requiere parámetros. Incluye el total de libros en la respuesta.

Parámetros

Ninguno requerido.

Respuestas
  • 200 Lista de todos los libros con el total
Respuesta de ejemplo
JSON 
{
  "total": 5,
  "libros": [
    {
      "id": 1,
      "nombre": "Cien años de soledad",
      "anioPublicacion": 1967
    }
  ]
}
GET /api/libros?nombre=texto Busca libros por nombre ▼
Descripción

Filtra libros cuyo nombre contenga el texto buscado. La búsqueda no distingue entre mayúsculas y minúsculas.

Parámetros de consulta
Nombre Tipo Requerido Descripción
nombre string Sí Texto a buscar en el título
Respuestas
  • 200 Libros que coinciden con la búsqueda
  • 404 No se encontraron libros con ese nombre
Respuesta 200
JSON · 200 OK
{
  "total": 1,
  "libros": [{
    "id": 4,
    "nombre": "1984",
    "autor": "George Orwell"
  }]
}
Respuesta 404
JSON · 404 Not Found
{
  "error": true,
  "mensaje": "No se encontraron libros con el nombre: \"xyz\""
}
GET /api/libros/:id Obtiene un libro por su ID ▼
Descripción

Ruta dinámica. El segmento :id es reemplazado por el número del libro que deseas obtener.

Parámetros de ruta
NombreTipoRequeridoDescripción
id integer Sí ID numérico del libro (1–5)
Respuestas
  • 200 Objeto del libro encontrado
  • 400 El ID no es un número válido
  • 404 No existe un libro con ese ID
Respuesta 200
JSON · 200 OK
{
  "id": 3,
  "nombre": "El principito",
  "anioPublicacion": 1943,
  "autor": "Antoine de Saint-Exupéry",
  "genero": "Fábula"
}
Respuesta 400
JSON · 400 Bad Request
{
  "error": true,
  "mensaje": "El parámetro id debe ser un número entero válido."
}
